01 · ANATOMY

Three columns. Zero terminal.

The PTY is still running underneath — it's just not the surface anymore. The conversation is the hero. Everything else collapses, expands, or disappears.

  • Sessions list with live status dots — running, waiting, idle
  • Conversation pane with editorial-grade typography (Source Serif 4 for titles)
  • Right rail tabs into Files, MCP servers, or Context — never hidden in JSON
  • Terminal lives behind a pill in the corner. One click reveals it as a drawer.
  • Composer mirrors Claude Code's modes (Ask · Edit · Plan · Auto) but as a real segmented control

02 · SETTINGS

No more ~/.claude/ JSON.

Every Claude Code config that today lives in dotfiles or environment variables gets a real, searchable, typed settings surface. Per-project overrides inherit from globals. Diff what you've changed at a glance.

  • Global → workspace → project → session inheritance, visualised
  • System prompt editor with version history and per-project overrides
  • MCP server registry — one-click connect, auth status, tool inventory
  • Approval rules with regex preview — "always allow npm test*" etc.
  • Skill library with auto-load conditions per project

03 · PRINCIPLES

What makes Atrium different.

I.

The terminal is a drawer, not a surface.

The PTY runs underneath. You can drop into it any time with ⌘⇧T. But you almost never have to — diffs render properly, permissions are modals, and tool calls collapse into cards. Cursor for PMs, but for code.

II.

Settings are first-class.

Every config that today lives in ~/.claude/ dotfiles gets a typed, searchable surface. Global → workspace → project → session inheritance, visualised. No more grep-and-pray.

III.

It wraps, never replaces.

Spawns the real claude CLI underneath. Your CLAUDE.md, skills, MCP servers, slash commands, hooks — all keep working. Atrium is the chrome, not the engine.

IV.

Lookout is the remote.

When you're at your desk, Atrium is the surface. When you're not, Lookout watches the same sessions from your phone. Same hook plumbing, same SQLite permission queue, same SSE bus. Siblings, not competitors.

PART TWO

The rest of the surface.

Eight more views — from first-launch to history search, from the terminal drawer to the dark theme. This is what the product actually looks like end-to-end.

04 · TERMINAL

When you do need it.

Hit ⌘⇧T or click the pill — the terminal slides up from below as a drawer. The conversation stays visible above it. Both panes scroll independently. ESC or click outside to dismiss.

  • Real PTY · xterm.js renderer · full ANSI support
  • Same session as the conversation above — they share state
  • Drag the divider to resize · double-click to maximise
  • Tab between multiple shells if you spawn them

05 · PLAN MODE

Think before you act.

Plan mode is read-only. Claude maps out the work as numbered steps with file targets and time estimates — but writes nothing until you approve. Useful for unfamiliar codebases and high-stakes refactors.

  • Read-only by default · no edits, no shell, no commits
  • Each step typed: read, trace, edit, test, commit
  • Edit individual steps inline · drag to reorder · delete to skip
  • Once approved, execution streams into the conversation pane
06 · MCP

A marketplace, not a JSON file.

Browse, search, and connect MCP servers without ever editing a config. Auth flows happen in-app. Tool inventories are visible before you connect. Your choices sync across machines.

  • One-click OAuth · no copy-pasting tokens into dotfiles
  • Per-server tool inventory and rate limit visibility
  • Workspace vs global scope · share a config with your team
  • "Bring your own" custom MCPs from URL or local binary

07 · HISTORY

Every session, searchable.

Full-text search across every transcript you've ever had. Filter by workspace, status, date, or starred. The first sentence of the latest turn shows in the preview — enough to remember what it was about without opening it.

  • Full-text search via SQLite FTS5 — millisecond latency
  • Group by day · keyboard navigable · J/K to move
  • Star sessions to keep them out of auto-archive
  • Export any session as Markdown, share, or fork from any turn
08 · DARK

Same soul. Less light.

Dark mode isn't just inverted colours. The greens shift toward emerald, the warm cream becomes warm graphite, and the serif headings stay — they read better on dark backgrounds than most teams realise.

  • → Background #0d0d0d · elevated #1a1a1a
  • → Auto-switches with system · or manual override
  • → Diffs use desaturated greens/reds for night-time reading

11 · ROADMAP

A weekend, not a year.

Scope this honestly — Atrium is a polish layer over an engine that already works. Ship the shell, then earn each feature.

v0.1 · WK1
Tauri shell · spawn the CLI · render conversation
Three-column layout. Read the JSONL transcript. Markdown rendering. Composer that pipes back to the CLI's stdin. No terminal yet — text only.
MVP
v0.2 · WK2
Diff rendering · permission modals · screenshot paste
The actual differentiation vs Clarc. Tool calls collapse into cards. Permission cards have proper buttons. Cmd+V on an image just works.
DELIGHT
v0.3 · WK3
Settings surface · MCP toggles · slash palette (⌘K)
First-class settings for everything in ~/.claude/. The MCP marketplace can be a stub that just lists what's installed.
MOAT
v0.4 · WK4
Terminal drawer · history search · dark theme
xterm.js drawer. SQLite FTS5 for transcript search. Dark theme as a polish ship.
SHIP

EARN
A note to self

This case study is a portfolio piece, not a build commitment. Clarc and Anthropic Desktop already cover most of this surface — the moat would be taste, not capability. The right next move is to lift Atrium's design language into Lookout, ship Cursor for PMs at the hackathon, and let this concept earn its v0.1 by getting a real user first.
