MCP marketplace

2026-05-04· 6 tickets· ~/projects/atrium/artifacts/wave-4-mcp-marketplace.md

Why now

Atrium currently inherits the user's installed MCP servers. To be product-friendly, we want a discoverable marketplace inside the app — installs in two clicks, surfaces capability before commitment.

Acceptance criteria

  • ✓List installed MCP servers and their tools, grouped by server.STO-2138
  • ✓Show tool count, last activity, and "in use by N skills" for each server.STO-2139
  • Browse a curated registry of community MCPs — read-only catalogue.STO-2140
  • One-tap install flow that writes to ~/.claude/mcp.json with explicit confirmation.STO-2141
  • Per-server permission preview — Bash, Edit, Read paths the MCP can touch.STO-2142
  • Telemetry off by default; explicit toggle for usage metrics.STO-2143

Out of scope

Publishing flow for community MCPs · server-side ratings · payment-gated MCPs.

Motion · six rules

shadcn easings: 150ms standard, 300ms emphasized. One thing moves at a time.

One thing breathes at a time.

Every motion answers a state question — where Claude is, what's selected, where a wave just went. If a motion doesn't answer one of those, it doesn't ship.

M-01
Breathing caret

The "›" inside the active todo's tick. Claude is mid-sentence. 1.8s ease-in-out, opacity 0.45→1.0.

›
M-02
Live pulse

Green dot on the live-chip. This wave has Claude attached. 2.4s ease, 5px max ring.

M-03
Cursor block

End of active sub-line, in the composer, in the terminal. 1.05s blink. Same dialect across surfaces.

›
M-04
Cell breathe

Background of the journey-ribbon current cell. 4.2s slow breathe. Tells you where you are without competing.

M-05
Pin glow (once)

Pin focus animation. Plays once on mount, sits still — no infinite ring. 700ms ease-out-expo.

M-06
Loop-back glide

Wave card slides stage→stage on cubic-bezier(0.16, 1, 0.3, 1). 480ms with mid-flight opacity dip.
